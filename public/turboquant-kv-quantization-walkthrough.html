TurboQuant: rotation, KV cache, and residual quantization

TurboQuant is a KV-cache compression method for long-context LLM inference. The core idea is to store keys and values in a low-bit form, keep the expensive full-precision reconstruction transient inside the attention computation, and use a rotation or projection trick so aggressive quantization damages the attention signal less.

Rotation: what is random, what is trained?

There are two different objects that can be easy to mix together: the model's normal projection weights and the extra transform used for quantization. The model projection weights are learned during pre-training. The TurboQuant-style rotation used to make KV vectors more quantization-friendly is not learned in the usual sense; it is typically sampled as a random orthogonal transform, such as a randomized Hadamard or random rotation, and then kept fixed.

Learned during pre-training

Transformer weights like Wq, Wk, and Wv are optimized by gradient descent.

hidden_state x Wk = key vector k
hidden_state x Wv = value vector v
trained model parameters

Fixed for quantization

The rotation or JL-style projection is chosen to spread outliers across coordinates before quantization. It is a mathematical preprocessing transform, not a new trainable layer in the base model.

rotated_k = R x k
R is sampled once, then reused
random orthogonal transform
Reconciliation. If a lecture sentence sounded like "the rotation matrix parameters are trained during pre-training," the most likely intended meaning is that the model's K/V-producing projection weights were trained during pre-training. The paper's "random rotation matrix" refers to the extra quantization transform, which is randomly generated and fixed.

Why rotate at all?

Quantization hates outliers. If one coordinate is huge and most coordinates are tiny, a low-bit grid spends too much of its range on that single coordinate and wipes out subtle structure elsewhere.

A random orthogonal rotation preserves distances and inner products in full precision, but redistributes energy across coordinates. After rotation, the same vector is less spiky and easier to quantize.

What does not live there?

The reconstructed floating-point K/V values do not need to be written back into the cache. Reconstructing them in HBM would defeat much of the memory-saving purpose.

The cache is long-lived. The dequantized values are short-lived. They exist just long enough for attention math.

Attention reads compressed K/V and reconstructs transiently

At a future token, the current query needs to attend over all previous keys and values. The attention kernel streams compressed K/V from HBM, decodes small tiles into registers or SRAM, performs floating-point dot products and weighted sums, then moves on.

1. Read

Quantized K/V blocks are loaded from HBM. This is the expensive traffic TurboQuant reduces.

HBM bandwidth saved

2. Decode inside kernel

Low-bit codes are dequantized in registers or SRAM-like on-chip storage. These FP values are working values, not a persistent cache.

transient FP math

3. Compute and discard

The kernel computes q·k, softmax, and the weighted sum over V. The reconstructed K/V tile is discarded after use.

no FP writeback
for each tile of cached tokens: packed_k, packed_v = read_from_HBM() k_tile, v_tile = dequantize_in_registers_or_SRAM(packed_k, packed_v) scores += q @ k_tile.T output += softmax(scores) @ v_tile discard(k_tile, v_tile)
This is the correct mental model: quantized K/V are stored in HBM; dequantized floating-point K/V are temporary inside the attention kernel; the temporary reconstructed values are not written back to HBM.

The two-stage quantization pipeline

The useful way to understand the MSE plus QJL design is as a base approximation plus an error-correction sketch. The first stage captures the easy, high-energy part of the vector. The second stage stores a very cheap summary of what the first stage missed.

1

Stage 1: MSE quantizer

The vector is quantized with parameters chosen to minimize mean squared error between the original vector and the reconstructed vector. This gives a strong low-bit approximation.

q_base = Quantize_MSE(x)
x_base = Dequantize(q_base)
residual r = x - x_base
2

Stage 2: 1-bit Quantized JL transform on the residual

The residual still contains information that can matter for attention. Instead of storing the full residual, a Johnson-Lindenstrauss-style random projection compresses it into a small sketch, then stores only the sign bits.

sketch = sign(S x r)
S = random projection / JL transform
store sketch with 1 bit per projected coordinate
3

During attention: combine both estimates

The base quantized vector gives the main approximation. The QJL residual sketch gives a cheap correction to preserve dot-product behavior, especially the angular or similarity information that attention scores care about.

attention score q·x ≈ q·x_base + correction_from_QJL_residual

Why a residual stage helps

A tiny base quantizer might get the broad vector direction right but lose smaller components. Those smaller components can change dot products across many cached tokens.

QJL uses randomness differently from scalar quantization: it compresses geometry. The sign of random projections can preserve similarity information surprisingly well for very little storage.

Base quantizer: stores approximate coordinates.
QJL residual: stores a compressed hint about the leftover geometry.

Precise takeaways

TopicClean mental model
Rotation matrixThe model's projection weights are trained during pre-training. The quantization rotation/JL transform is randomly generated or structured-random and fixed; it is used to spread outliers and preserve geometry before low-bit storage.
KV cache contentsThe persistent HBM cache stores quantized K/V codes plus any needed scales or metadata, not full dequantized FP K/V.
Attention-time computationCompressed K/V are read from HBM and dequantized transiently inside the attention kernel, typically in registers or on-chip memory, then used for floating-point score and value computations.
No writebackThe temporary dequantized K/V tiles are discarded after the dot product or weighted-sum tile. Writing them back to HBM would largely erase the memory benefit.
Two-stage quantizationThe MSE quantizer gives a base approximation. The 1-bit QJL residual stores a low-cost geometric correction for the quantization error.
A concise one-line summary: TurboQuant compresses the KV cache by storing low-bit, geometry-aware representations of K and V; the attention kernel briefly reconstructs or estimates what it needs for dot products, but the long-lived memory footprint stays compressed.
